# CC — Boot Context

## Identity
机器人公司 CTO，全职负责技术架构与团队技术方向。量化交易为个人兴趣项目（统计套利 / 配对交易）。

## Environment
- 全职：机器人公司，负责系统架构、算法、嵌入式、云平台
- 副项目：个人量化交易系统，独立开发，对接 Interactive Brokers

## Languages
中文为主要沟通语言，代码和文档用英文。

## Tech Stack
- **核心语言:** C++, Python
- **机器人:** ROS, 嵌入式系统
- **云平台:** 有经验，具体栈视项目而定
- **量化:** Python, pandas, numpy, statsmodels, ib_insync
- **AI 工具:** Claude Code, Codex CLI, Gemini CLI（重度用户）

## Capability Profile
- 算法能力: **最强** — 核心竞争力
- 系统架构: **强** — 能设计复杂系统
- 业务逻辑: **中上** — 理解到位
- 前后端细节: **弱** — 不熟悉前后端实现细节
- AI 工具: **重度用户** — 日常深度使用多个 AI 平台

**核心优势: 算法 + 系统架构能力，善于用 AI 工具放大产出。**

## Communication Rules
- 中文沟通，简洁直接
- 需要时再详细解释，默认精简
- 结构化输出：表格、列表、清晰分层
- 注重逻辑：用最精简清晰的逻辑表达
- 分析问题先从运行日志/实证出发，不要只看代码猜测
- 没有看完全部相关信息之前不要开始分析

## Known Blind Spots (for AI to monitor)
- **过度优化倾向** — 容易在已经够用的方案上继续优化。AI 应提醒"当前方案是否已满足需求"
- **前后端细节不熟** — 涉及前后端实现时，AI 应主动给出更详细的说明和注意事项
